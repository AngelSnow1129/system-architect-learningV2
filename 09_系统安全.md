## **信息安全基础知识**

*   **信息安全包括5个基本要素**：机密性、完整性、可用性、可控性与可审查性。
    1.  **机密性**：确保信息不暴露给未授权的实体或进程。
    2.  **完整性**：只有得到允许的人才能修改数据，并且能够判别出数据是否已被篡改。
    3.  **可用性**：得到授权的实体在需要时可访问数据，即攻击者不能占用所有的资源而阻碍授权者的工作。
    4.  **可控性**：可以控制授权范围内的信息流向及行为方式。
    5.  **可审查性**：对出现的信息安全问题提供调查的依据和手段。
*   **信息安全的范围包括**：设备安全、数据安全、内容安全和行为安全。
    1.  **设备安全**：信息系统安全的首要问题，是信息系统安全的物质基础，包括设备的稳定性、可靠性、可用性。
    2.  **数据安全**：确保数据免受未授权的泄露、篡改和毁坏，包括数据的秘密性、完整性、可用性。
    3.  **内容安全**：信息安全在政治、法律、道德层次上的要求，包括信息内容政治上健康、符合国家法律法规、符合道德规范。
    4.  **行为安全**：确保信息系统的行为安全，才能最终确保系统的信息安全。行为安全的特性包括行为的秘密性、完整性、可控性。
*   **信息的存储安全**包括信息使用的安全、系统安全监控、计算机病毒防治、数据的加密和防止非法的攻击等。
    1.  **信息使用的安全**：包括用户的标识与验证、用户存取权限限制。
    2.  **系统安全监控**：建立一套安全监控系统，全面监控系统活动，并随时检查系统使用情况。还应建立完善的审计系统和日志管理系统。
    3.  **计算机病毒防治**：计算机网络服务器必须加装网络病毒自动检测系统，并定期更新。
*   **网络安全**
    *   **网络安全隐患**：物理安全性、软件安全漏洞、不兼容使用安全漏洞、选择合适的安全哲理。
    *   **网络安全威胁**：非授权的访问、信息泄露或丢失、破坏数据完整性、拒绝服务攻击、利用网络传播病毒。
    *   **安全措施的目标**：访问控制、认证、完整性、审计、保密。

## **信息安全系统的组成框架**

*   **技术体系**：从实现技术上来看，信息安全系统涉及以下技术：
    1.  **基础安全设备**：包括密码芯片、加密卡、身份识别卡等，以及物理环境保障技术。
    2.  **计算机网络安全**：信息在网络传输过程中的安全防范，涉及物理隔离、防火墙、加密传输、认证、数字签名、VPN、病毒防范等。
    3.  **操作系统安全**：操作系统的无错误配置、无漏洞、无后门等。安全机制包括标识与鉴别、访问控制、最小特权管理等。
    4.  **数据库安全**：分为数据库管理系统安全和数据库应用系统安全，涉及物理和逻辑完整性、元素安全性、访问控制、身份认证等。
    5.  **终端安全设备**：如电话密码机、传真密码机、异步数据密码机等。

## **信息安全技术**

### **加密技术**
一个密码系统，通常简称为**密码体制 (Cryptosystem)**，由五部分组成：
1.  **明文空间M**：全体明文的集合。
2.  **密文空间C**：全体密文的集合。
3.  **密钥空间K**：全体密钥的集合。其中每一个密钥 K 均由加密密钥 Ke 和解密密钥 Kd 组成，即 k = <ke, kd>。
4.  **加密算法E**：一组由M至C的加密变换。
5.  **解密算法D**：一组由C到M的解密变换。

*   对于明文空间M中的每一个明文M，加密算法E在密钥Ke的控制下将明文M加密成密文C： **C = E(M, Ke)**
*   而解密算法D在密钥Kd的控制下将密文C解密出同一明文M： **M = D(C, Kd) = D(E(M, Ke), Kd)**

### **对称加密技术**
*   数据的加密和解密的密钥（密码）是相同的，属于不公开密钥加密算法。
*   **缺点**：加密强度不高（因为密钥位数少），且密钥分发困难。
*   **优点**：加密速度快，适合加密大数据。
*   **常见的对称密钥加密算法**：
    *   **DES**：替换 + 移位、56位密钥、64位数据块、速度快，密钥易产生。
    *   **3DES**：三重DES，两个56位密钥K1、K2。
        *   加密：K1加密 → K2解密 → K1加密
        *   解密：K1解密 → K2加密 → K1解密
    *   **AES**：美国联邦政府采用的区块加密标准，用来替代原先的DES。
    *   **RC-5**：RSA数据安全公司的很多产品都使用了RC-5。
    *   **IDEA**：128位密钥，64位数据块，比DES的加密性好，对计算机功能要求相对低。

### **非对称加密技术**
*   数据的加密和解密的密钥是不同的，分为**公钥**和**私钥**。是公开密钥加密算法。
*   **缺点**：加密速度慢。
*   **优点**：安全性高，不容易破解。
*   **原理**：发送者发送数据时，使用接收者的**公钥**作加密密钥，接收者使用自己的**私钥**作解密密钥。安全性更高，因为无需传输解密密钥。

![](./images/09_系统安全/09_系统安全-2.png)

*   **常见的非对称加密算法**：RSA、ElGamal、ECC（椭圆曲线算法）、背包算法、Rabin、D-H等。

### **数字信封**
*   **原理**：将对称加密技术和非对称加密技术组合使用。
*   **过程**：发送方将数据用**对称密钥**加密传输，而将该对称密钥用**接收方公钥**加密发送给对方。接收方收到后，用自己的**私钥**解密出对称密钥，再用对称密钥解密数据得到原文。
*   **作用**：解决了对称密钥的安全传输问题。

### **信息摘要**
*   一段数据的特征信息，由**哈希函数**生成。当数据发生改变，信息摘要也会发生改变。
*   **特点**：
    *   产生固定长度的信息摘要。
    *   任何不同的输入数据，都会产生不同的信息摘要。
    *   单向性（不可逆）。
*   **信息摘要算法**：MD5（128位输出）、SHA-1（160位输出，安全性更高）。

### **数字签名**
*   唯一标识一个发送方。
*   **过程**：发送者发送数据时，使用发送者的**私钥**进行加密（签名），接收者收到数据后，只能使用发送者的**公钥**进行解密（验证）。这样就能唯一确定发送方，并保证信息未被篡改和发送行为的不可否认性。但无法保证机密性。

![](./images/09_系统安全/09_系统安全-3.png)

### **公钥基础设施（PKI）**
*   以不对称密钥加密技术为基础，以数据机密性、完整性、身份认证和行为不可抵赖性为安全目的，来实施和提供安全服务的具有普适性的安全基础设施。
    1.  **数字证书**：由可信任的权威机构签署的信息集合，用于确保公钥及其与用户绑定关系的安全。如x.509证书必须包含版本号、序列号、签名算法标识符、认证机构、有效期限、主题信息、认证机构的数字签名、公钥信息等。
    2.  **签证机构（CA）**：负责签发、管理和撤销证书。是所有注册用户所信赖的权威机构。

### **访问控制**
*   访问控制是指主体依据某些控制策略或权限对客体本身或是其资源进行的不同授权访问。包括3个要素：**主体、客体和控制策略**。
*   **访问控制实现技术**：
    1.  **访问控制矩阵（ACM）**：通过矩阵形式表示访问控制规则。主体作为行，客体作为列。

    | | file1 | file2 | file3 |
    | :--- | :---: | :---: | :---: |
    | **User1** | rw | | rw |
    | **User2** | r | rwx | x |
    | **User3** | x | r | |

    2.  **访问控制表（ACL）**：最流行、使用最多的技术。每个客体有一个访问控制表，记录有权访问该客体的主体信息（按列保存矩阵）。
    3.  **能力表**：每个主体有一个能力表，记录该主体对系统中每个客体的访问权限信息（按行保存矩阵）。

    ![](./images/09_系统安全/09_系统安全-4.png)

    4.  **授权关系表**：每一行就是一个非空元素，是某个主体对应某个客体的访问权限信息。

## **信息安全的抗攻击技术**

*   为对抗攻击，密钥生成需考虑：增大密钥空间、选择强钥、密钥的随机性。
*   **拒绝服务（DoS）攻击**：使合法用户不能及时得到资源或服务。
    *   **外部用户攻击模式**：消耗资源、破坏或更改配置信息、物理破坏或改变网络部件、利用服务程序中的处理错误使服务失效。
    *   **分布式拒绝服务（DDoS）攻击**：利用多个被控制的计算机同时向一个目标发起攻击。
    *   **防御方式**：加强数据包特征识别、设置防火墙监视端口、统计通信数据量、修正系统漏洞。

### **ARP欺骗**
*   **正常ARP原理**：主机A广播ARP请求，询问主机B的MAC地址。主机B单播ARP响应，告知自己的MAC地址。主机A缓存此映射。
*   **ARP欺骗原理**：攻击者C向主机A发送伪造的ARP响应，声称主机B的IP地址对应的是C的MAC地址。主机A更新缓存，导致发往B的数据被发送到C。

![](./images/09_系统安全/09_系统安全-5.png)

*   **防范措施**：静态绑定ARP表、使用ARP服务器、双向绑定、使用ARP防护软件。

### **DNS欺骗**
*   攻击者冒充域名服务器，将查询的IP地址指向攻击者的IP地址，从而将用户流量重定向到恶意网站。

![](./images/09_系统安全/09_系统安全-6.png)

*   **DNS欺骗检测**：
    1.  **被动监听检测**：监听DNS请求和应答，若一个请求对应多个不同结果的应答，则可能存在欺骗。
    2.  **虚假报文探测**：向非DNS服务器发送请求，若收到应答则存在攻击。
    3.  **交叉检查查询**：收到DNS应答后，反向查询IP地址对应的域名是否与原域名一致。

### **IP欺骗**
*   **原理**：伪造IP包的源地址，冒充另一台主机与目标主机进行通信。
*   **防范**：删除`hosts.equiv`和`.rhosts`文件，修改`inetd.conf`禁用RPC，设置防火墙过滤源地址为内部IP的外部报文。

### **端口扫描**
*   尝试与目标主机的某些端口建立连接，以判断端口是否开放。
*   **扫描原理分类**：
    1.  **全TCP连接**：完成三次握手建立标准TCP连接。
    2.  **半打开式扫描（SYN扫描）**：只发送SYN包，根据响应（SYN+ACK或RST）判断端口状态，不完成连接。
    3.  **FIN扫描**：发送FIN包，关闭的端口会回复RST，活动的端口无响应。更隐蔽。
    4.  **第三方扫描（代理扫描）**：利用被入侵的第三方主机（肉鸡）进行扫描。

### **强化TCP/IP堆栈抵御攻击**
1.  **同步包风暴（SYN Flooding）**：发送大量伪造的TCP连接请求（SYN包），但不完成三次握手，耗尽服务器的半连接队列资源。
2.  **ICMP攻击**：利用ICMP协议的特点进行攻击，如“Ping of Death”攻击，发送超大ICMP包导致系统崩溃。
3.  **SNMP攻击**：利用SNMP协议控制网络设备，重定向通信流或断开连接。

## **信息安全的保证体系和评估方法**

*   **GB17859-1999标准规定了计算机系统安全保护能力的五个等级**：
    1.  **第一级：用户自主保护级**：通过隔离用户与数据，实现自主访问控制。
    2.  **第二级：系统审计保护级**：实现更细粒度的自主访问控制，通过登录规程、审计和资源隔离使用户对行为负责。
    3.  **第三级：安全标记保护级**：在二级基础上，提供安全策略模型、数据标记和强制访问控制。
    4.  **第四级：结构化保护级**：建立在形式化安全策略模型之上，将自主和强制访问控制扩展到所有主体与客体，并考虑隐蔽通道。
    5.  **第五级：访问验证保护级**：满足访问监控器需求，访问监控器仲裁所有访问，自身抗篡改且足够小以便分析测试。

### **安全风险管理**
*   **风险评估实施前应考虑**：确定范围、目标，建立组织结构、方法，并获得管理者批准。
*   **风险评估基本要素**：脆弱性、资产、威胁、风险和安全措施。
*   **风险计算过程**：
    1.  识别信息资产并赋值。
    2.  分析威胁并对发生可能性赋值。
    3.  识别资产脆弱性并对严重程度赋值。
    4.  根据威胁和脆弱性计算安全事件发生的可能性。
    5.  结合资产重要性和事件可能性，计算风险值。

## **网络安全技术**
*   **防火墙**：在内部网络和外部因特网之间的安全防护措施。
    *   **网络级防火墙**：层次低，效率高，使用包过滤和状态监测。对应用和用户透明。
    *   **应用级防火墙**：层次高，效率低，检查网络包内容数据，安全强度高。
*   **入侵检测系统（IDS）**：防火墙之后的第二道屏障，作为补充。
    *   **原理**：监听和分析系统/用户行为，与知识库匹配，检测入侵行为，记录并上报。
    *   **部署位置**：挂接在所有关注流量都必须流经的链路上，如尽可能靠近攻击源或受保护资源。
*   **入侵防御系统（IPS）**：提前发现并主动阻断入侵行为。
*   **杀毒软件**：用于检测和解决计算机病毒。
*   **蜜罐系统**：伪造一个网络引诱黑客攻击，以了解其攻击手段，从而优化安全系统。

## **网络安全协议**

物理层主要使用物理手段，其他层依靠协议保证安全。

![](./images/09_系统安全/09_系统安全-7.png)

*   **SSL协议**：安全套接字协议，加强Web安全传输（HTTP/HTTPS），端口号为443。
*   **SSH协议**：安全外壳协议，加强Telnet/FTP安全的传输协议。
*   **SET协议**：安全电子交易协议，主要应用于B2C电子商务中保障支付信息的安全。
*   **Kerberos协议**：网络身份认证协议，基于信任第三方，在开放网络中进行身份认证。
*   **PGP协议**：使用RSA公钥证书进行身份认证，使用IDEA进行数据加密，使用MD5进行数据完整性验证。
