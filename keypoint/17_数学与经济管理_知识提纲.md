### 重点1：最小生成树 ⭐⭐⭐⭐⭐
**难度**：★★★☆☆
**考试频率**：高

**问题描述：** ⭐⭐⭐⭐⭐
```
给定一个带权连通图，找到一棵生成树，使得所有边的权值之和最小

应用场景：
- 网络布线（最小成本）
- 管道铺设
- 电路设计
- 道路建设
```

**求解算法：** ⭐⭐⭐⭐⭐

**Prim算法（普里姆算法）：**
```
思想：从一个顶点开始，逐步扩展生成树

步骤：
1. 选择任意一个顶点作为起点，加入生成树
2. 在所有连接"树内顶点"和"树外顶点"的边中，
   选择权值最小的边
3. 将该边和对应的树外顶点加入生成树
4. 重复步骤2-3，直到所有顶点都在生成树中

特点：
- 适合稠密图
- 时间复杂度：O(n²)
```

**Kruskal算法（克鲁斯卡尔算法）：**
```
思想：按边的权值从小到大选择边

步骤：
1. 将所有边按权值从小到大排序
2. 依次选择权值最小的边
3. 如果该边的两个顶点不在同一连通分量中，则加入生成树
4. 如果该边会形成环，则跳过
5. 重复步骤2-4，直到选择了n-1条边

特点：
- 适合稀疏图
- 时间复杂度：O(e log e)
```

**考试解题方法：** ⭐⭐⭐⭐⭐

**方法1：贪心法（推荐）**
```
步骤：
1. 从任意顶点开始
2. 每次选择与当前树相连的最小权值边
3. 不能形成环
4. 重复直到所有顶点都连接

技巧：
- 在图上直接标记已选择的边
- 用不同颜色区分已选和未选
- 记录每次选择的边和权值
```

**典型例题：** ⭐⭐⭐⭐⭐

**例题1：** 9栋楼房之间敷设管道，使各楼连通且总成本最低。

```
解题步骤：

1. 列出所有边及权值（从小到大）：
   1-2: 1
   2-3: 1
   3-4: 1
   1-5: 2
   2-6: 2
   ...

2. 使用贪心法选择：
   第1条：选1-2（权值1）
   第2条：选2-3（权值1）
   第3条：选3-4（权值1）
   第4条：选1-5（权值2）
   第5条：选2-6（权值2）
   ...
   
3. 选择n-1条边（9个顶点需要8条边）

4. 计算总权值
```

**例题2：** 8口海上油井铺设输油管道，从海岸开始经1#油井连接所有油井。

```
解题步骤：

1. 起点：海岸到1#油井（5海里）

2. 从1#开始，选择最小权值边：
   1#-5#: 0.5
   
3. 从{1#, 5#}选择最小权值边：
   5#-8#: 0.8
   
4. 继续选择最小权值边，不形成环：
   4#-1#: 0.9
   5#-6#: 0.9
   6#-7#: 0.6
   7#-3#: 1.9（或其他）
   3#-2#: 0.9
   
5. 计算总长度：
   5 + 0.5 + 0.8 + 0.9 + 0.9 + 0.6 + 1.9 + 0.9 = 10.5
   （实际需要根据具体选择计算）
```

**快速技巧：** ⭐⭐⭐⭐⭐
```
1. 优先选择权值最小的边
2. 确保不形成环（已连接的顶点不再连接）
3. n个顶点需要n-1条边
4. 在图上直接标记，避免遗漏
```

### 重点2：最短路径 ⭐⭐⭐⭐⭐
**难度**：★★★☆☆
**考试频率**：高

**问题描述：** ⭐⭐⭐⭐⭐
```
给定一个带权图，找到两个顶点之间权值和最小的路径

应用场景：
- 导航系统
- 网络路由
- 物流配送
```

**Dijkstra算法：** ⭐⭐⭐⭐⭐

**算法思想：**
```
从起点开始，逐步确定到各顶点的最短路径

步骤：
1. 初始化：起点距离为0，其他顶点距离为∞
2. 选择未访问顶点中距离最小的顶点v
3. 更新v的邻接顶点的距离：
   如果 dist[v] + edge(v,u) < dist[u]
   则 dist[u] = dist[v] + edge(v,u)
4. 标记v为已访问
5. 重复步骤2-4，直到所有顶点都访问或到达终点

特点：
- 不能处理负权边
- 时间复杂度：O(n²)
```

**考试解题方法：** ⭐⭐⭐⭐⭐

**方法1：表格法（推荐）**
```
步骤：
1. 画表格，列出所有顶点
2. 初始化起点距离为0，其他为∞
3. 每轮选择最小距离的未访问顶点
4. 更新其邻接顶点的距离
5. 标记该顶点为已访问
6. 重复直到到达终点
```

**典型例题：** ⭐⭐⭐⭐⭐

**例题：** 从A到F的最短距离

```
给定距离表：
     B   C   D   E   F
A   11  16  24  36  54
B       13  16  21  29
C           14  17  22
D               14  17
E                   15

解题步骤：

初始状态：
A=0, B=∞, C=∞, D=∞, E=∞, F=∞

第1轮：从A出发
A=0 (已访问)
更新：B=11, C=16, D=24, E=36, F=54

第2轮：选择B(11)
B=11 (已访问)
更新：C=min(16, 11+13)=16
      D=min(24, 11+16)=24
      E=min(36, 11+21)=32
      F=min(54, 11+29)=40

第3轮：选择C(16)
C=16 (已访问)
更新：D=min(24, 16+14)=24
      E=min(32, 16+17)=32
      F=min(40, 16+22)=38

第4轮：选择D(24)
D=24 (已访问)
更新：E=min(32, 24+14)=32
      F=min(38, 24+17)=38（不变）

第5轮：选择E(32)
E=32 (已访问)
更新：F=min(38, 32+15)=38（不变）

最终：A到F的最短距离 = 38
路径：A → C → F (16+22=38)
```

**答案：38**

**快速技巧：** ⭐⭐⭐⭐⭐
```
1. 每轮选择距离最小的未访问顶点
2. 更新时比较"原距离"和"经过当前顶点的距离"
3. 取较小值
4. 可以提前结束（到达终点即可）
```

### 重点3：网络最大流量 ⭐⭐⭐⭐
**难度**：★★★★☆
**考试频率**：中

**问题描述：** ⭐⭐⭐⭐
```
给定一个网络（有向图），每条边有容量限制，
求从源点到汇点的最大流量

应用场景：
- 交通运输
- 网络带宽
- 供水系统
```

**Ford-Fulkerson算法：** ⭐⭐⭐⭐

**算法思想：**
```
不断寻找增广路径，直到找不到为止

步骤：
1. 初始化流量为0
2. 寻找从源点到汇点的增广路径
3. 找到路径上的最小容量（瓶颈）
4. 沿路径增加流量
5. 更新剩余容量
6. 重复步骤2-5，直到找不到增广路径
```

**考试解题方法：** ⭐⭐⭐⭐⭐

**方法：枚举路径法**
```
步骤：
1. 列出所有从源点到汇点的路径
2. 对每条路径，找出最小容量（瓶颈）
3. 分配流量，更新剩余容量
4. 重复寻找路径，直到无法增加流量
5. 累加所有路径的流量
```

**典型例题：** ⭐⭐⭐⭐⭐

**例题：** 从节点①到节点⑥的最大运输能力

```
解题步骤：

路径1：① → ② → ⑤ → ⑥
瓶颈：min(6, 7, 21) = 6
流量：6

路径2：① → ③ → ⑤ → ⑥
瓶颈：min(10, 14, 21-6) = 10
流量：10

路径3：① → ④ → ⑥
瓶颈：min(10, 5) = 5
流量：5

最大流量 = 6 + 10 + 5 = 21

注意：
- 每条边的容量是有限的
- 已使用的容量要从剩余容量中扣除
- 可能需要多次尝试不同路径组合
```

**快速技巧：** ⭐⭐⭐⭐
```
1. 优先选择容量大的路径
2. 注意边的容量限制
3. 已分配的流量要从容量中扣除
4. 可能需要回溯调整
```

### 重点4：线性规划 ⭐⭐⭐⭐
**难度**：★★★★☆
**考试频率**：中

**问题描述：** ⭐⭐⭐⭐
```
在一组线性约束条件下，求线性目标函数的极值

组成：
1. 目标函数：需要最大化或最小化的线性函数
2. 约束条件：一组线性不等式或等式
3. 非负约束：变量通常要求非负
```

**标准形式：** ⭐⭐⭐⭐⭐
```
目标函数：
max(或min) Z = c₁x₁ + c₂x₂ + ... + cₙxₙ

约束条件：
a₁₁x₁ + a₁₂x₂ + ... + a₁ₙxₙ ≤ b₁
a₂₁x₁ + a₂₂x₂ + ... + a₂ₙxₙ ≤ b₂
...
aₘ₁x₁ + aₘ₂x₂ + ... + aₘₙxₙ ≤ bₘ

非负约束：
x₁, x₂, ..., xₙ ≥ 0
```

**考试解题方法：** ⭐⭐⭐⭐⭐

**方法：顶点枚举法**
```
步骤：
1. 建立数学模型（目标函数和约束条件）
2. 求出约束条件的交点（顶点）
3. 将顶点坐标代入目标函数
4. 比较各顶点的目标函数值
5. 选择最优解
```

**典型例题：** ⭐⭐⭐⭐⭐

**例题：** 生产I、II两种产品，求最大利润

```
已知条件：
- 产品I利润：9万元/吨
- 产品II利润：12万元/吨
- 原材料限制见表格

建立模型：
设生产产品I为x吨，产品II为y吨

目标函数：
max P = 9x + 12y

约束条件：
x + y ≤ 4      (甲原料)
4x + 3y ≤ 12   (乙原料)
x + 3y ≤ 6     (丙原料)
x ≥ 0, y ≥ 0

求解：
找出约束条件的交点：

交点1：(0, 0)
P = 9×0 + 12×0 = 0

交点2：(0, 2)  [x=0代入x+3y≤6]
P = 9×0 + 12×2 = 24

交点3：(3, 1)  [4x+3y=12与x+y=4的交点]
P = 9×3 + 12×1 = 39

交点4：(3, 0)  [y=0代入4x+3y≤12]
P = 9×3 + 12×0 = 27

最优解：x=3, y=1
最大利润：39万元
```

**答案：39万元**

**快速技巧：** ⭐⭐⭐⭐
```
1. 最优解通常在可行域的顶点处
2. 求出所有约束条件的交点
3. 检查交点是否满足所有约束
4. 将可行交点代入目标函数
5. 比较选择最优值
```

**快速记忆法：** ⭐⭐⭐⭐⭐

**最小生成树：**
```
"贪心选最小，不能成环，n个点n-1边"
```

**最短路径：**
```
"Dijkstra贪心法，每轮选最小，更新邻接点"
```

**最大流量：**
```
"找路径，看瓶颈，累加流量"
```

**线性规划：**
```
"建模型，找顶点，代入比较"
```

**易错点提醒：** ⭐⭐⭐⭐⭐
1. 最小生成树：n个顶点需要n-1条边
2. 最短路径：每轮选择未访问的最小距离顶点
3. 最大流量：注意容量限制，已用容量要扣除
4. 线性规划：检查交点是否满足所有约束条件
5. 最小生成树不能形成环
6. Dijkstra算法不能处理负权边
