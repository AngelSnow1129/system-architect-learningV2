## 16.1 人工智能与机器学习

### 16.1.1 人工智能基础
- **AI发展历程**

  - 连接主义（神经网络）
  - 行为主义（强化学习）

### 16.1.3 深度学习

  - RNN/LSTM：循环神经网络（序列）
  - Transformer：注意力机制（NLP）
  - GAN：生成对抗网络
- **预训练模型**
  - BERT、GPT系列
  - ResNet、VGG
  - 迁移学习
- **应用领域**
  - 计算机视觉
  - 自然语言处理
  - 语音识别
  - 推荐系统

## 16.14 典型例题精讲 ⭐⭐⭐⭐⭐

### 例题1：机器学习分类 ⭐⭐⭐⭐⭐
以下哪种不属于监督学习？
A. 分类  B. 回归  C. 聚类  D. 决策树

**解析**：
```
监督学习：有标签数据
- 分类：预测离散值（如垃圾邮件分类）
- 回归：预测连续值（如房价预测）
- 决策树：分类或回归算法

无监督学习：无标签数据
- 聚类：数据分组（如客户细分）
- 降维：特征提取（如PCA）
- 关联规则：发现关联（如购物篮分析）
```
**答案**：C

### 例题2：大数据5V ⭐⭐⭐⭐⭐
大数据的5V特征不包括？
A. Volume  B. Velocity  C. Variety  D. Visibility

**解析**：
```
大数据5V特征：
1. Volume（体量）：数据规模大
2. Velocity（速度）：处理速度快
3. Variety（多样）：数据类型多
4. Value（价值）：价值密度低
5. Veracity（真实性）：数据质量

Visibility（可见性）不是5V之一
```
**答案**：D

### 例题3：云服务模型 ⭐⭐⭐⭐⭐
用户只需关注应用程序，不需要管理基础设施和平台，这是？
A. IaaS  B. PaaS  C. SaaS  D. FaaS

**解析**：
```
IaaS：用户管理OS、中间件、应用
- 示例：AWS EC2、阿里云ECS

PaaS：用户管理应用程序
- 示例：Google App Engine、Heroku

SaaS：用户只使用应用，无需管理
- 示例：Office365、Salesforce

FaaS：函数即服务，Serverless
- 示例：AWS Lambda、阿里云函数计算
```
**答案**：C

### 例题4：区块链共识 ⭐⭐⭐⭐⭐
比特币使用的共识算法是？
A. PoS  B. PoW  C. DPoS  D. PBFT

**解析**：
```
PoW（Proof of Work）工作量证明：
- 比特币使用
- 通过计算哈希值竞争记账权
- 能耗高，安全性高
- 性能低（约7 TPS）

其他共识算法：
- PoS：以太坊2.0
- DPoS：EOS
- PBFT：联盟链（如Hyperledger Fabric）
```
**答案**：B

### 例题5：5G应用场景 ⭐⭐⭐⭐⭐
5G的三大应用场景不包括？
A. eMBB  B. URLLC  C. mMTC  D. IoT

**解析**：
```
5G三大应用场景：
1. eMBB（Enhanced Mobile Broadband）
   - 增强移动宽带
   - 高速率：Gbps级
   - 应用：4K/8K视频、VR/AR

2. URLLC（Ultra-Reliable Low-Latency Communications）
   - 超可靠低延迟通信
   - 低延迟：1ms
   - 应用：自动驾驶、远程手术

3. mMTC（Massive Machine Type Communications）
   - 海量机器类通信
   - 大连接：百万/km²
   - 应用：智慧城市、工业物联网

IoT是应用领域，不是5G的三大场景
```
**答案**：D

### 例题6：IoT架构 ⭐⭐⭐⭐
IoT架构中负责数据采集的是？
A. 感知层  B. 网络层  C. 平台层  D. 应用层

**解析**：
```
IoT四层架构：
1. 感知层：数据采集
   - 传感器、RFID、摄像头
   - 负责感知物理世界

2. 网络层：数据传输
   - 通信协议、网关
   - 负责数据传输

3. 平台层：数据处理
   - 数据存储、处理、分析
   - 设备管理、规则引擎

4. 应用层：业务应用
   - 用户界面、业务逻辑
   - 面向最终用户
```
**答案**：A

### 例题7：深度学习架构 ⭐⭐⭐⭐
以下哪种神经网络架构最适合处理图像？
A. RNN  B. LSTM  C. CNN  D. Transformer

**解析**：
```
神经网络架构及应用：

CNN（卷积神经网络）：
- 适用：图像处理
- 特点：局部连接、权值共享

RNN/LSTM（循环神经网络）：
- 适用：序列数据
- 特点：记忆历史信息
- 应用：语音识别、时间序列

Transformer：
- 适用：自然语言处理
- 特点：注意力机制
- 应用：机器翻译、文本生成
```
**答案**：C

### 例题8：边缘计算 ⭐⭐⭐⭐
边缘计算相比云计算的主要优势是？
A. 计算能力更强  B. 存储容量更大  C. 延迟更低  D. 成本更低

**解析**：
```
边缘计算特点：
1. 低延迟：数据在边缘处理，无需上传云端
2. 本地处理：减少带宽消耗
3. 隐私保护：敏感数据不离开本地
4. 实时响应：适合实时性要求高的场景

云计算特点：
1. 计算能力强：集中式大规模计算
2. 存储容量大：海量数据存储
3. 弹性伸缩：按需分配资源
4. 成本优化：规模效应

边缘计算主要优势：低延迟、实时性
```
**答案**：C

## 16.16 易错点提醒 ⭐⭐⭐⭐⭐

### 易错点1：监督学习与无监督学习
```
❌ 错误：聚类是监督学习
✅ 正确：聚类是无监督学习
区分：
- 监督学习：有标签（分类、回归）
- 无监督学习：无标签（聚类、降维）
```

### 易错点2：大数据5V
```
❌ 错误：5V包括Visibility（可见性）
✅ 正确：5V是Volume、Velocity、Variety、Value、Veracity
记忆：体速多价真
```

### 易错点3：云服务模型管理责任
```
❌ 错误：PaaS用户需要管理操作系统
✅ 正确：
- IaaS：用户管理OS、中间件、应用
- PaaS：用户只管理应用
- SaaS：用户无需管理任何基础设施
```

### 易错点4：IoT架构层次
```
❌ 错误：网络层负责数据采集
✅ 正确：
- 感知层：数据采集
- 网络层：数据传输
- 平台层：数据处理
- 应用层：业务应用
```

### 易错点5：区块链共识算法
```
❌ 错误：以太坊使用PoW
✅ 正确：
- 以太坊1.0：PoW
- 以太坊2.0：PoS
- 比特币：PoW
- EOS：DPoS
```

### 易错点6：5G三大场景
```
❌ 错误：IoT是5G的三大场景之一
✅ 正确：
- 5G三大场景：eMBB、URLLC、mMTC
- IoT是应用领域，不是场景
- mMTC支持IoT应用
```

### 易错点7：边缘计算与云计算
```
❌ 错误：边缘计算完全替代云计算
✅ 正确：
- 边缘计算：低延迟、本地处理
- 云计算：大规模计算、海量存储
- 两者互补，不是替代关系
```

### 易错点8：深度学习架构应用
```
❌ 错误：RNN适合处理图像
✅ 正确：
- CNN：图像处理
- RNN/LSTM：序列数据（语音、文本）
- Transformer：自然语言处理
```

## 16.18 考试答题技巧 ⭐⭐⭐⭐⭐

### 技巧1：识别技术类型
```
关键词识别：
- "标签"、"分类"、"回归" → 监督学习
- "聚类"、"降维" → 无监督学习
- "奖励"、"策略" → 强化学习
- "TB"、"PB"、"实时" → 大数据
- "虚拟机"、"容器" → 云计算
- "传感器"、"RFID" → 物联网
- "去中心化"、"不可篡改" → 区块链
```

### 技巧2：云服务模型判断
```
看用户管理内容：
- 管理OS、中间件、应用 → IaaS
- 只管理应用 → PaaS
- 无需管理 → SaaS

看服务提供内容：
- 提供虚拟机、存储 → IaaS
- 提供开发平台 → PaaS
- 提供应用软件 → SaaS
```

### 技巧3：区块链共识算法
```
看代表项目：
- 比特币 → PoW
- 以太坊2.0 → PoS
- EOS → DPoS
- 联盟链 → PBFT

看特点：
- 高能耗、低性能 → PoW
- 低能耗、中性能 → PoS
- 高性能、部分去中心 → PBFT
```

### 技巧4：5G场景匹配
```
看应用需求：
- 高速率（Gbps） → eMBB
- 低延迟（1ms） → URLLC
- 大连接（百万/km²） → mMTC

看具体应用：
- 4K/8K、VR/AR → eMBB
- 自动驾驶、远程手术 → URLLC
- 智慧城市、工业物联网 → mMTC
```

### 技巧5：排除法
```
遇到不确定的题目：
1. 排除明显错误的选项
2. 关注关键词和特征
3. 回忆相关概念和分类
4. 选择最符合定义的选项
```
