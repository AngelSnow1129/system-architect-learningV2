## 🎯 核心知识点框架

### 1. 数据库系统基础 ⭐⭐

#### 1.1 基本概念
**数据（Data）：** 描述事物的符号记录

**数据库（DB）：** 长期存储、有组织、可共享的大量数据集合

**数据库特征：**
- 数据按一定模型组织
- 可为各种用户共享
- 冗余度较小
- 数据独立性较高
- 易扩展

#### 1.2 数据库系统（DBS）组成
```
数据库系统（DBS）
├── 数据库（DB）
├── 硬件（存储设备）
├── 软件（OS、DBMS、应用程序）
└── 人员（DBA、程序员、用户）
```

**DBMS功能：**
- 数据定义
- 数据库操作
- 数据库运行管理
- 数据存储管理
- 数据库建立和维护

#### 1.3 三级模式结构 ⭐⭐⭐⭐⭐
```
外模式（视图）
    ↓ 外模式-模式映像
模式（基本表）
    ↓ 模式-内模式映像
内模式（物理存储文件）
```

**对应关系：**
- 外模式 ↔ 视图
- 模式 ↔ 基本表
- 内模式 ↔ 存储文件

**数据独立性：** ⭐⭐⭐⭐⭐
- **物理独立性**：通过修改模式-内模式映像实现
- **逻辑独立性**：通过修改外模式-模式映像实现

### 3. 数据模型 ⭐⭐⭐⭐

#### 3.1 模型分类
- **概念模型**：用户角度，E-R模型
- **关系模型**：二维表形式
- **网状模型**：网状联系
- **面向对象模型**：对象为单位

**数据模型三要素：**
1. 数据结构
2. 数据操作
3. 数据约束条件

#### 3.2 E-R模型 ⭐⭐⭐⭐⭐

**基本元素：**
- 椭圆：属性
- 矩形：实体
- 菱形：联系

**实体类型：**
- **强实体**：独立存在
- **弱实体**：依赖强实体存在

**属性分类：**
- 简单属性 vs 复合属性
- 单值属性 vs 多值属性
- NULL属性
- 派生属性

**联系类型：** ⭐⭐⭐⭐⭐
- **1:1（一对一）**
- **1:N（一对多）**
- **M:N（多对多）**

#### 3.3 E-R模型转关系模型 ⭐⭐⭐⭐⭐

**转换规则：**

**1:1联系：**
- 可放到任意一端实体中作为属性
- 也可转换为独立关系模式

**1:N联系：**
- 可单独作为关系模式
- 也可在N端加入1端实体的主键

**M:N联系：** ⭐⭐⭐⭐⭐
- **必须**作为独立关系模式
- 主键是M端和N端的联合主键

### 4. 关系代数 ⭐⭐⭐⭐

#### 4.1 基本运算

**并（∪）：**
- 两表所有记录合并
- 相同记录只显示一次

**交（∩）：**
- 两表相同的记录

**差（-）：**
- S1 - S2：S1有而S2没有的记录

**笛卡尔积（×）：** ⭐⭐⭐⭐
- 属性列：S1 + S2的所有列
- 记录数：S1记录数 × S2记录数
- S1每条记录与S2所有记录组合

**投影（π）：**
- 选择某些列
- 可用数字表示列号

**选择（σ）：**
- 选择某些行（记录）
- 按条件筛选

**自然连接（⋈）：** ⭐⭐⭐⭐⭐
- 显示全部属性列，相同列只显示一次
- 显示属性相同且值相同的记录
- 连接条件：公共属性值相等

**计算公式：**
- 自然连接后属性数 = 两表属性总数 - 公共属性数
- 例：R(A,B,C,D,E) ⋈ S(B,C,F,G) = 7列（A,B,C,D,E,F,G）

### 5. 函数依赖与范式 ⭐⭐⭐⭐⭐

#### 5.1 函数依赖

**基本概念：**
- 给定X，能唯一确定Y，则Y依赖于X（X→Y）

**部分函数依赖：**
- (A,B)可确定C，A也可确定C
- C部分依赖于(A,B)

**传递函数依赖：**
- A→B，B→C，则A→C
- 注意：A和B不等价时才是传递

#### 5.2 Armstrong公理系统 ⭐⭐⭐

**基本规则：**
- **自反律**：Y⊆X⊆U，则X→Y
- **增广律**：X→Y，则XZ→YZ
- **传递律**：X→Y，Y→Z，则X→Z

**推导规则：**
- **合并规则**：X→Y，X→Z，则X→YZ
- **伪传递率**：X→Y，WY→Z，则XW→Z
- **分解规则**：X→Y，Z⊆Y，则X→Z

#### 5.3 键与约束 ⭐⭐⭐⭐

**键的类型：**
- **超键**：能唯一标识的属性集
- **候选键**：超键去掉冗余属性
- **主键**：任选一个候选键
- **外键**：其他表的主键
- **主属性**：候选键内的属性
- **非主属性**：不在候选键内的属性

**完整性约束：**
- **实体完整性**：主键不能为空，不能重复
- **参照完整性**：外键必须是已存在的主键值或为空
- **用户自定义完整性**：自定义约束表达式

#### 5.4 范式 ⭐⭐⭐⭐⭐

**第一范式（1NF）：**
- 每个分量必须是不可分的数据项
- 表中不允许有小表

**第二范式（2NF）：** ⭐⭐⭐⭐
- 满足1NF
- 每个非主属性完全函数依赖于候选码
- 消除非主属性对码的部分依赖

**第三范式（3NF）：** ⭐⭐⭐⭐⭐
- 满足2NF
- 不存在非主属性对码的传递依赖
- 消除传递依赖

**BC范式（BCNF）：** ⭐⭐⭐⭐
- 满足3NF
- 消除主属性对码的部分和传递依赖
- 每个依赖的左边决定因素都包含候选键

**范式关系：**
```
1NF ⊃ 2NF ⊃ 3NF ⊃ BCNF
```

#### 5.5 候选键求法 ⭐⭐⭐⭐⭐

**步骤：**
1. 找出从未在依赖右边出现的属性（必在候选键中）
2. 以该属性为基础，根据依赖集扩展
3. 看能否遍历所有属性
4. 无法遍历的属性加入候选键

**示例：**
- U={A,B,C,D}，F={AB→C, C→D, D→B}
- A只在左边出现，必在候选键中
- A+ = {A}
- AB+ = {A,B,C,D}
- AC+ = {A,C,D,B}
- 候选键：AB和AC

### 6. 模式分解 ⭐⭐⭐⭐⭐

#### 6.1 保持函数依赖分解 ⭐⭐⭐⭐

**定义：** 分解后的关系模式保持原依赖集不变

**判断方法：**
- 简单方法：看每个依赖的左右属性是否都在同一分解模式中
- 通用方法：使用算法逐个检查依赖

#### 6.2 无损分解 ⭐⭐⭐⭐⭐

**定义：** 分解后能还原出原关系模式

**两个关系模式判断定理：**
- 分解p={R1, R2}具有无损连接的充要条件：
  - (R1∩R2) → (R1-R2) 或
  - (R1∩R2) → (R2-R1)

**三个及以上关系模式：** 使用表格法

**表格法步骤：** ⭐⭐⭐⭐⭐
1. 画表格，行为分解后的关系，列为所有属性
2. 属性在该关系中标√，不在标×
3. 根据函数依赖，若某行在左边属性列都是√，则右边属性列也改为√
4. 重复步骤3，直到不再变化
5. 若某行全为√，则是无损分解

### 7. 并发控制 ⭐⭐⭐⭐⭐

#### 7.1 事务ACID特性 ⭐⭐⭐⭐⭐

**原子性（Atomicity）：**
- 要么全做，要么全不做

**一致性（Consistency）：**
- 事务后数据保持一致
- 例：转账，A减少=B增加

**隔离性（Isolation）：**
- 事务更新对其他事务不可见
- 不同事务互不干涉

**持久性（Durability）：**
- 事务操作结果是持久的

#### 7.2 并发问题 ⭐⭐⭐⭐⭐

**丢失更新：**
- T1修改A并写回
- T2也修改A并写回
- T2覆盖T1的更新

**不可重复读：**
- T2读A
- T1修改A并写回
- T2再读A，数据不一致

**读脏数据：**
- T1修改A
- T2读A
- T1回滚
- T2读到的是脏数据

#### 7.3 封锁协议 ⭐⭐⭐⭐⭐

**锁的类型：**
- **X锁（排它锁/写锁）**：只允许加锁事务读取和修改
- **S锁（共享锁/读锁）**：只允许加锁事务读取，其他事务可加S锁

**一级封锁协议：**
- 修改前加X锁，事务结束释放
- 解决：丢失更新

**二级封锁协议：**
- 一级基础上，读前加S锁，读完即释放
- 解决：丢失更新、读脏数据

**三级封锁协议：** ⭐⭐⭐⭐⭐
- 一级基础上，读前加S锁，事务结束释放
- 解决：丢失更新、读脏数据、不可重复读

#### 8.1 数据库安全

### 9. SQL语言 ⭐⭐⭐⭐⭐

#### 9.1 数据定义语言（DDL）

**创建表：**
```sql
CREATE TABLE 表名(
    列名 数据类型 [约束],
    ...
    PRIMARY KEY(主键列),
    FOREIGN KEY(外键列) REFERENCES 表名(列名)
);
```

**修改表：**
```sql
ALTER TABLE 表名 ADD 列名 数据类型;
ALTER TABLE 表名 DROP COLUMN 列名;
```

**删除表：**
```sql
DROP TABLE 表名;
```

**创建索引：**
```sql
CREATE [UNIQUE] INDEX 索引名 ON 表名(列名);
```

**创建视图：**
```sql
CREATE VIEW 视图名 AS SELECT语句;
```

#### 9.2 数据查询语言（DQL）⭐⭐⭐⭐⭐

**基本查询：**
```sql
SELECT [ALL|DISTINCT] 列名
FROM 表名
[WHERE 条件]
[GROUP BY 列名 [HAVING 条件]]
[ORDER BY 列名 [ASC|DESC]];
```

**关键字说明：**
- **DISTINCT**：去除重复记录
- **WHERE**：筛选条件
- **GROUP BY**：分组
- **HAVING**：分组后的筛选条件
- **ORDER BY**：排序（ASC升序，DESC降序）

**聚合函数：**
- COUNT()：计数
- SUM()：求和
- AVG()：平均值
- MAX()：最大值
- MIN()：最小值

**字符串匹配：**
- **LIKE**：模糊匹配
- **%**：匹配任意多个字符
- **_**：匹配任意一个字符

**集合运算：**
- **UNION**：并集
- **INTERSECT**：交集
- **EXCEPT**：差集

**连接查询：**
```sql
-- 自然连接
SELECT * FROM R, S WHERE R.B=S.B AND R.C=S.C;

-- 内连接
SELECT * FROM R INNER JOIN S ON R.B=S.B;

-- 外连接
SELECT * FROM R LEFT JOIN S ON R.B=S.B;
```

#### 9.3 数据操作语言（DML）

**插入：**
```sql
INSERT INTO 表名 VALUES(值1, 值2, ...);
INSERT INTO 表名(列1, 列2) VALUES(值1, 值2);
```

**更新：**
```sql
UPDATE 表名 SET 列名=值 WHERE 条件;
```

**删除：**
```sql
DELETE FROM 表名 WHERE 条件;
```

#### 9.4 SQL与关系代数对应 ⭐⭐⭐⭐⭐

**投影（π）：**
```sql
SELECT 列名 FROM 表名;
```

**选择（σ）：**
```sql
SELECT * FROM 表名 WHERE 条件;
```

**自然连接（⋈）：**
```sql
SELECT * FROM R, S WHERE R.公共列=S.公共列;
```

**组合表达式：**
```
π列名(σ条件(R⋈S))
↓
SELECT 列名 FROM R, S WHERE 连接条件 AND 选择条件;
```

## 🔍 典型例题分析

### 例题1：三级模式
**题目：** 视图、基本表、存储文件分别对应哪个模式？

**分析：**
- 视图 → 外模式
- 基本表 → 模式
- 存储文件 → 内模式

**答案：** 外模式、模式、内模式

### 例题2：E-R转关系
**题目：** 学生选课，一个学生选多门课，一门课被多个学生选，如何转换？

**分析：**
- 学生和课程是M:N联系
- 必须独立成表
- 主键是(学号, 课程号)

**答案：**
- 学生(学号, 姓名, ...)
- 课程(课程号, 课程名, ...)
- 选课(学号, 课程号, 成绩)

### 例题3：候选键求解
**题目：** U={A,B,C,D}，F={AB→C, C→D, D→B}，求候选键？

**分析：**
1. A只在左边，必在候选键中
2. A+ = {A}，不能遍历
3. AB+ = {A,B,C,D}，能遍历
4. AC+ = {A,C,D,B}，能遍历

**答案：** AB和AC

### 例题4：范式判断
**题目：** 学生(学号, 姓名, 系号, 系名)，F={学号→姓名, 学号→系号, 系号→系名}，属于哪个范式？

**分析：**
- 候选键：学号
- 姓名、系号完全依赖于学号，满足2NF
- 系名传递依赖于学号（学号→系号→系名），不满足3NF

**答案：** 2NF

### 例题5：无损分解
**题目：** R(A,B,C)，F={A→B, B→C}，分解为R1(A,B)和R2(B,C)，是否无损？

**分析：**
- R1∩R2 = {B}
- R1-R2 = {A}
- R2-R1 = {C}
- B→C成立（即(R1∩R2)→(R2-R1)）

**答案：** 是无损分解

### 例题6：SQL语句
**题目：** 查询每个学生的平均成绩，要求平均成绩>60

**SQL：**
```sql
SELECT 学号, AVG(成绩) AS 平均成绩
FROM 成绩表
GROUP BY 学号
HAVING AVG(成绩) > 60;
```

**要点：**
- 分组用GROUP BY
- 分组后筛选用HAVING
- 聚合函数AVG

## ⚠️ 易错点提醒

1. 候选码求解时，L类和N类必须包含
2. 部分依赖是针对复合候选码的
3. 传递依赖要求中间属性不是候选码的子集
4. 2NF无部分依赖，3NF无传递依赖
5. BCNF要求每个决定因素都包含候选码
6. 无损分解：公共属性必须能决定某个关系的非公共属性
7. SQL中GROUP BY后筛选用HAVING，不是WHERE
8. 主键不能为NULL，外键可以为NULL
9. 内连接只返回匹配的行，外连接返回所有行
10. 事务的隔离级别从低到高：读未提交、读已提交、可重复读、串行化
