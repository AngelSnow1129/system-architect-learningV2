## 🖥️ 计算机硬件

### 流水线性能计算 ⭐⭐⭐⭐⭐

```
流水线周期 = max(各段执行时间)

流水线执行时间 = 第1条指令执行时间 + (指令条数 - 1) × 流水线周期
               = Σ(各段时间) + (n - 1) × 流水线周期

吞吐率 = 指令条数 / 流水线执行时间

加速比 = 不使用流水线时间 / 使用流水线时间
       = (n × Σ各段时间) / [Σ各段时间 + (n-1) × 流水线周期]

最大加速比 = Σ各段时间 / 流水线周期
```

**记忆口诀**："周期取最长，首条全相加，后续乘周期"

## 💾 操作系统

### 分页存储地址转换 ⭐⭐⭐⭐⭐

```
逻辑地址 = 页号 × 页面大小 + 页内偏移

物理地址 = 物理页号 × 页面大小 + 页内偏移

页号 = 逻辑地址 / 页面大小（取整数部分）

页内偏移 = 逻辑地址 % 页面大小（取余数部分）
```

**记忆口诀**："除得页号，余得偏移"

### 多级索引文件大小 ⭐⭐⭐⭐⭐

```
直接索引：
文件大小 = 直接索引块数 × 块大小

一级间接索引：
文件大小 = 块大小 × (块大小 / 地址长度)

二级间接索引：
文件大小 = 块大小 × (块大小 / 地址长度)²

总文件大小 = 直接索引大小 + 一级索引大小 + 二级索引大小 + ...
```

**记忆口诀**："直接乘块，一级平方，二级立方"

## 🌐 计算机网络

### 子网划分 ⭐⭐⭐⭐⭐

```
子网数 = 2^n（n为借用的主机位数）

每个子网的主机数 = 2^m - 2（m为剩余的主机位数）
                  减2：网络地址和广播地址

子网掩码计算：
原掩码 + 借用位数对应的值

网络地址：IP地址 AND 子网掩码

广播地址：网络地址 + (2^m - 1)

主机范围：网络地址+1 到 广播地址-1
```

**记忆口诀**："借用定子网，剩余定主机，主机数减二"

## 🗄️ 数据库系统

### 候选码求解 ⭐⭐⭐⭐⭐

```
属性分类：
L类：只在依赖左边出现
R类：只在依赖右边出现
N类：不在任何依赖中出现
LR类：既在左边又在右边出现

候选码组成：
必须包含：L类 + N类
可能包含：LR类的部分或全部
不会包含：R类

验证方法：
计算候选码的闭包，看是否能推出所有属性
```

**记忆口诀**："L和N必含，LR可能含，R类不会含"

### 范式判断 ⭐⭐⭐⭐⭐

```
1NF：每个分量都是不可分的数据项

2NF：满足1NF，且非主属性完全函数依赖于候选码
    （无部分依赖）

3NF：满足2NF，且非主属性不传递依赖于候选码
    （无传递依赖）

BCNF：满足3NF，且每个决定因素都包含候选码
```

**记忆口诀**："1NF无小表，2NF无部分，3NF无传递，BCNF决定因素含候选码"

## 📊 系统可靠性

### 可靠性基本公式 ⭐⭐⭐⭐⭐

```
MTBF = MTTF + MTTR

可用性 A = MTTF / (MTTF + MTTR)
         = MTTF / MTBF
         = 1 - MTTR/MTBF

其中：
MTTF：平均无故障时间
MTTR：平均修复时间
MTBF：平均故障间隔时间
```

**记忆口诀**："MTBF等于MTTF加MTTR，可用性等于MTTF除MTBF"

### 串并联系统可靠性 ⭐⭐⭐⭐⭐

```
串联系统：
R_串 = R₁ × R₂ × R₃ × ... × Rₙ

并联系统（失效概率法）：
F_并 = (1-R₁) × (1-R₂) × ... × (1-Rₙ)
R_并 = 1 - F_并

三模冗余（TMR）：
R_TMR = 3R² - 2R³
```

**记忆口诀**："串联相乘，并联失效乘，三模冗余3R²减2R³"

### 可用性等级 ⭐⭐⭐⭐⭐

```
年停机时间 = 365 × 24 × 60 × (1 - 可用性) 分钟

99%（2个9）：年停机 3.65天
99.9%（3个9）：年停机 8.76小时
99.99%（4个9）：年停机 52.56分钟
99.999%（5个9）：年停机 5.26分钟
```

**记忆口诀**："2个9天，3个9小时，4个9分钟"

## 📅 项目管理

### 关键路径法 ⭐⭐⭐⭐⭐

```
最早开始时间（ES）：
ES = max(所有前置活动的EF)

最早结束时间（EF）：
EF = ES + 活动持续时间

最迟结束时间（LF）：
LF = min(所有后续活动的LS)

最迟开始时间（LS）：
LS = LF - 活动持续时间

总浮动时间：
总浮动时间 = LS - ES
或
总浮动时间 = LF - EF
或
总浮动时间 = 关键路径时长 - 本活动所在路径时长

自由浮动时间：
自由浮动时间 = min(紧后活动的ES) - 本活动的EF
```

**记忆口诀**："顺推ES和EF，逆推LF和LS，总浮动看路径差"

## 🔢 数据结构与算法

### 时间复杂度 ⭐⭐⭐⭐⭐

```
常见时间复杂度（从优到劣）：
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)

加法规则：
T(n) = T₁(n) + T₂(n) = O(max(f(n), g(n)))

乘法规则：
T(n) = T₁(n) × T₂(n) = O(f(n) × g(n))
```

**记忆口诀**："加法取最大，乘法直接乘"

### 二叉树性质 ⭐⭐⭐⭐⭐

```
性质1：第i层最多有2^(i-1)个节点

性质2：深度为k的二叉树最多有2^k - 1个节点

性质3：n₀ = n₂ + 1
      （叶子节点数 = 度为2的节点数 + 1）

性质4：完全二叉树深度 = ⌊log₂n⌋ + 1

性质5：完全二叉树中，节点i（从1开始）：
      父节点：⌊i/2⌋
      左孩子：2i
      右孩子：2i + 1
```

**记忆口诀**："叶子等于度2加1"

## 📐 数学与经济管理

### 最小生成树 ⭐⭐⭐⭐⭐

```
Prim算法：
1. 选择任意顶点作为起点
2. 选择与当前树相连的最小权值边
3. 重复直到所有顶点都在树中

Kruskal算法：
1. 将所有边按权值从小到大排序
2. 依次选择权值最小的边
3. 如果不形成环，则加入生成树
4. 重复直到选择n-1条边

n个顶点需要n-1条边
```

**记忆口诀**："贪心选最小，不能成环，n个点n-1边"

### 最短路径（Dijkstra） ⭐⭐⭐⭐⭐

```
算法步骤：
1. 初始化：起点距离为0，其他为∞
2. 选择未访问顶点中距离最小的顶点v
3. 更新v的邻接顶点的距离：
   如果 dist[v] + edge(v,u) < dist[u]
   则 dist[u] = dist[v] + edge(v,u)
4. 标记v为已访问
5. 重复2-4直到所有顶点都访问
```

**记忆口诀**："每轮选最小，更新邻接点"

### 线性规划 ⭐⭐⭐⭐

```
标准形式：
目标函数：max(或min) Z = c₁x₁ + c₂x₂ + ... + cₙxₙ

约束条件：
a₁₁x₁ + a₁₂x₂ + ... + a₁ₙxₙ ≤ b₁
a₂₁x₁ + a₂₂x₂ + ... + a₂ₙxₙ ≤ b₂
...

非负约束：x₁, x₂, ..., xₙ ≥ 0

求解方法：
1. 求出约束条件的交点（顶点）
2. 将顶点代入目标函数
3. 比较选择最优值
```

**记忆口诀**："建模型，找顶点，代入比较"

## 🔄 单位换算

### 时间单位 ⭐⭐⭐⭐⭐

```
1秒(s) = 10³ 毫秒(ms)
1秒(s) = 10⁶ 微秒(μs)
1秒(s) = 10⁹ 纳秒(ns)
1秒(s) = 10¹² 皮秒(ps)
```

### 存储单位 ⭐⭐⭐⭐⭐

```
1 KB = 2¹⁰ B = 1024 B
1 MB = 2²⁰ B = 1024 KB
1 GB = 2³⁰ B = 1024 MB
1 TB = 2⁴⁰ B = 1024 GB
```
